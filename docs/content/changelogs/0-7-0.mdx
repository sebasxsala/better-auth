# Better Auth Ruby v0.7.0

The Ruby package set is now released at `0.7.0` across core and the supported adapters/plugins.

```ruby title="Gemfile"
gem "better_auth", "0.7.0"
```

## OpenAPI Support

OpenAPI is now marked as supported for the Ruby package. The generator includes upstream `v1.6.9` base-route parity for public paths, richer request and response schemas, documented `/ok` and `/error` routes, plugin endpoint metadata checks, and Scalar reference HTML configuration parity.

## Security And Protocol Hardening

- OAuth client-secret verification now uses constant-time comparison for encrypted and custom-hashed storage modes.
- OAuth refresh token rotation rejects tokens presented by the wrong authenticated client.
- API-key sessions expose `tokenFingerprint` instead of storing raw API keys in session state.
- SCIM generated provider tokens are hashed by default.
- Stripe webhooks now require a verified Stripe webhook constructor before processing payloads.

## Adapter And Plugin Parity

This release tightens MongoDB, Redis, Rails, Hanami, Sinatra, SSO, OAuth Provider, Passkey, API Key, SCIM, and Stripe behavior against upstream Better Auth `v1.6.9` semantics. It also expands package-specific tests and release verification coverage for the changed routes, adapters, middleware, and docs.

See the repository `CHANGELOG.md` and package changelogs for the detailed per-gem notes.
