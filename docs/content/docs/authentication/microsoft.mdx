---
title: Microsoft Entra ID
description: Microsoft Entra ID provider setup and usage in Better Auth Ruby.
---

Microsoft Entra ID is supported by the Ruby port through `BetterAuth::SocialProviders.microsoft_entra_id`.

## Callback URL

Configure the OAuth app with this redirect URL for local development:

```txt
http://localhost:3000/api/auth/callback/microsoft-entra-id
```

For production, use your application domain with the same path:

```txt
https://your-domain.com/api/auth/callback/microsoft-entra-id
```

## Configure The Provider

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  social_providers: {
    microsoft_entra_id: BetterAuth::SocialProviders.microsoft_entra_id(
      client_id: ENV.fetch("MICROSOFT_CLIENT_ID"),
      client_secret: ENV.fetch("MICROSOFT_CLIENT_SECRET"),
          tenant_id: ENV.fetch("MICROSOFT_TENANT_ID", "common")
    )
  }
)
```

The Ruby factory is `microsoft_entra_id` and uses `tenant_id`, defaulting to `common`.

## Start Sign-In

```ruby title="server.rb"
response = auth.api.sign_in_social(
  body: {
    provider: "microsoft-entra-id",
    callbackURL: "/dashboard",
    errorCallbackURL: "/login",
    disableRedirect: true
  }
)

redirect_url = response.fetch(:url)
```

In a Rack response, redirect the user to `redirect_url`. The OAuth callback is handled by Better Auth at `/api/auth/callback/microsoft-entra-id`.
