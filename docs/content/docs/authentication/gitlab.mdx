title: GitLab
description: GitLab provider setup and usage in Better Auth Ruby.
---

GitLab is supported by the Ruby port through `BetterAuth::SocialProviders.gitlab`.

## Callback URL

Configure the OAuth app with this redirect URL for local development:

```txt
http://localhost:3000/api/auth/callback/gitlab
```

For production, use your application domain with the same path:

```txt
https://your-domain.com/api/auth/callback/gitlab
```

## Configure The Provider

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  social_providers: {
    gitlab: BetterAuth::SocialProviders.gitlab(
      client_id: ENV.fetch("GITLAB_CLIENT_ID"),
      client_secret: ENV.fetch("GITLAB_CLIENT_SECRET")
    )
  }
)
```

Use this provider for GitLab.com or pass custom endpoints for self-managed GitLab setups when needed.

## Start Sign-In

```ruby title="server.rb"
response = auth.api.sign_in_social(
  body: {
    provider: "gitlab",
    callbackURL: "/dashboard",
    errorCallbackURL: "/login",
    disableRedirect: true
  }
)

redirect_url = response.fetch(:url)
```

In a Rack response, redirect the user to `redirect_url`. The OAuth callback is handled by Better Auth at `/api/auth/callback/gitlab`.
