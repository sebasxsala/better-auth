---
title: GitHub
description: GitHub provider setup and usage in Better Auth Ruby.
---

GitHub is supported by the Ruby port through `BetterAuth::SocialProviders.github`.

## Callback URL

Configure the OAuth app with this redirect URL for local development:

```txt
http://localhost:3000/api/auth/callback/github
```

For production, use your application domain with the same path:

```txt
https://your-domain.com/api/auth/callback/github
```

## Configure The Provider

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  social_providers: {
    github: BetterAuth::SocialProviders.github(
      client_id: ENV.fetch("GITHUB_CLIENT_ID"),
      client_secret: ENV.fetch("GITHUB_CLIENT_SECRET")
    )
  }
)
```

GitHub can return profile and email data through the built-in Ruby provider factory.

## Start Sign-In

```ruby title="server.rb"
response = auth.api.sign_in_social(
  body: {
    provider: "github",
    callbackURL: "/dashboard",
    errorCallbackURL: "/login",
    disableRedirect: true
  }
)

redirect_url = response.fetch(:url)
```

In a Rack response, redirect the user to `redirect_url`. The OAuth callback is handled by Better Auth at `/api/auth/callback/github`.
