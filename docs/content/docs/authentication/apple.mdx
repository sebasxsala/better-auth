---
title: Apple
description: Apple provider setup and usage in Better Auth Ruby.
---

Apple is supported by the Ruby port through `BetterAuth::SocialProviders.apple`.

## Callback URL

Configure the OAuth app with this redirect URL for local development:

```txt
http://localhost:3000/api/auth/callback/apple
```

For production, use your application domain with the same path:

```txt
https://your-domain.com/api/auth/callback/apple
```

## Configure The Provider

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  social_providers: {
    apple: BetterAuth::SocialProviders.apple(
      client_id: ENV.fetch("APPLE_CLIENT_ID"),
      client_secret: ENV.fetch("APPLE_CLIENT_SECRET")
    )
  }
)
```

Apple OAuth uses the same callback path as the upstream route surface. Apple client secrets are JWTs; generate and rotate them in your app or credentials system.

## Start Sign-In

```ruby title="server.rb"
response = auth.api.sign_in_social(
  body: {
    provider: "apple",
    callbackURL: "/dashboard",
    errorCallbackURL: "/login",
    disableRedirect: true
  }
)

redirect_url = response.fetch(:url)
```

In a Rack response, redirect the user to `redirect_url`. The OAuth callback is handled by Better Auth at `/api/auth/callback/apple`.
