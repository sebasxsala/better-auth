Expose this app as an OpenID Connect provider.

<Callout type="info">
This page documents the current Ruby port behavior. Ruby uses snake_case option names and `auth.api` method names; HTTP paths and JSON keys keep the upstream wire shape where implemented.
</Callout>

<Callout type="warn">
Some UI or provider-specific setup from upstream is app-provided in Ruby. The protocol endpoints below are implemented, but you must supply your own pages, callbacks, client objects, or provider metadata where noted by the option names.
</Callout>

## Configure

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  plugins: [
    BetterAuth::Plugins.oidc_provider(consent_page: "/oauth/consent")
  ]
)
```

## Usage

```ruby title="server.rb"
metadata = auth.api.get_open_id_config

client = auth.api.register_o_auth_application(
  body: {
    client_name: "Example Client",
    redirect_uris: ["https://client.example.com/callback"]
  }
)

token = auth.api.o_auth2_token(
  body: {
    grant_type: "authorization_code",
    code: params[:code],
    client_id: client[:client_id]
  }
)
```

## Routes

| Method | Path | Ruby API method |
| --- | --- | --- |
| `GET` | `/.well-known/openid-configuration` | `auth.api.get_open_id_config` |
| `POST` | `/oauth2/register` | `auth.api.register_o_auth_application` |
| `GET` | `/oauth2/authorize` | `auth.api.o_auth2_authorize` |
| `POST` | `/oauth2/consent` | `auth.api.o_auth_consent` |
| `POST` | `/oauth2/token` | `auth.api.o_auth2_token` |
| `GET` | `/oauth2/userinfo` | `auth.api.o_auth2_user_info` |

## Options

Current Ruby options accepted by `BetterAuth::Plugins.oidc_provider`:

- `scopes`
- `metadata`
- `default_scope`
- `store`
- `code_expires_in`
- `consent_page`
