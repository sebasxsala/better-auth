Stripe subscriptions and webhook handling.

<Callout type="info">
This page documents the current Ruby port behavior. Ruby uses snake_case option names and `auth.api` method names; HTTP paths and JSON keys keep the upstream wire shape where implemented.
</Callout>

<Callout type="warn">
Some UI or provider-specific setup from upstream is app-provided in Ruby. The protocol endpoints below are implemented, but you must supply your own pages, callbacks, client objects, or provider metadata where noted by the option names.
</Callout>

## Configure

```ruby title="config/auth.rb"
require "better_auth"
require "better_auth/stripe"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  plugins: [
    BetterAuth::Plugins.stripe(stripe_api_key: ENV.fetch("STRIPE_SECRET_KEY"), stripe_webhook_secret: ENV.fetch("STRIPE_WEBHOOK_SECRET"))
  ]
)
```

Enable subscription routes by passing `subscription.enabled` and a plan list:

```ruby title="config/auth.rb"
BetterAuth::Plugins.stripe(
  stripe_api_key: ENV.fetch("STRIPE_SECRET_KEY"),
  stripe_webhook_secret: ENV.fetch("STRIPE_WEBHOOK_SECRET"),
  subscription: {
    enabled: true,
    plans: [
      {
        name: "pro",
        price_id: "price_monthly",
        annual_discount_price_id: "price_yearly",
        limits: { projects: 10 },
        free_trial: { days: 14 }
      }
    ]
  }
)
```

For organization seat billing, enable the organization plugin and configure `seat_price_id`:

```ruby title="config/auth.rb"
plugins: [
  BetterAuth::Plugins.organization,
  BetterAuth::Plugins.stripe(
    stripe_api_key: ENV.fetch("STRIPE_SECRET_KEY"),
    stripe_webhook_secret: ENV.fetch("STRIPE_WEBHOOK_SECRET"),
    organization: { enabled: true },
    subscription: {
      enabled: true,
      plans: [
        {
          name: "team",
          price_id: "price_team_base",
          seat_price_id: "price_team_seat",
          line_items: [{ price: "price_metered_events" }],
          proration_behavior: "create_prorations"
        }
      ],
      authorize_reference: ->(_data, _ctx) { true }
    }
  )
]
```

## Usage

```ruby title="server.rb"
checkout = auth.api.upgrade_subscription(headers: { "cookie" => request.env["HTTP_COOKIE"] }, body: { plan: "pro", successUrl: "https://app.example.com/success", cancelUrl: "https://app.example.com/billing" })
portal = auth.api.create_billing_portal(headers: { "cookie" => request.env["HTTP_COOKIE"] }, body: { returnUrl: "https://app.example.com/billing" })
```

## Routes

| Method | Path | Ruby API method |
| --- | --- | --- |
| `POST` | `/subscription/upgrade` | `auth.api.upgrade_subscription` |
| `POST` | `/subscription/cancel` | `auth.api.cancel_subscription` |
| `POST` | `/subscription/restore` | `auth.api.restore_subscription` |
| `GET` | `/subscription/list` | `auth.api.list_active_subscriptions` |
| `POST` | `/subscription/billing-portal` | `auth.api.create_billing_portal` |
| `POST` | `/stripe/webhook` | `auth.api.stripe_webhook` |

## Options

Current Ruby options accepted by `BetterAuth::Plugins.stripe`:

- `create_customer_on_sign_up`
- `stripe_api_key` (defaults to `ENV["STRIPE_SECRET_KEY"]` when omitted)
- `stripe_webhook_secret`
- `stripe_client`
- `client`
- `subscription.enabled`
- `subscription.plans`
- `subscription.authorize_reference`
- `subscription.get_checkout_session_params`
- `subscription.on_subscription_complete`
- `subscription.on_subscription_created`
- `subscription.on_subscription_update`
- `subscription.on_subscription_cancel`
- `subscription.on_subscription_deleted`
- `on_event`
- `organization.enabled`
- `organization.get_customer_create_params`
- `organization.on_customer_create`

Plan entries support `name`, `price_id`, `lookup_key`, `annual_discount_price_id`, `annual_discount_lookup_key`, `limits`, `free_trial`, `seat_price_id`, `line_items`, and `proration_behavior`.
