---
title: MS SQL
description: Use Better Auth Ruby with Microsoft SQL Server.
---

Better Auth Ruby includes a direct MSSQL adapter for Rack applications backed by
Microsoft SQL Server.

<Callout type="info">
MSSQL support uses Sequel internally for safer parameter binding over TinyTDS.
This does not expose a public Sequel adapter; configure
`BetterAuth::Adapters::MSSQL` directly.
</Callout>

## Install Dependencies

```ruby title="Gemfile"
gem "better_auth"
gem "sequel"
gem "tiny_tds"
```

```bash title="Terminal"
bundle install
```

## Configure The Adapter

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL"),
  database: BetterAuth::Adapters::MSSQL.new(
    url: ENV.fetch("DATABASE_URL")
  )
)
```

You may also pass an existing Sequel database object when your app owns
connection management.

```ruby title="config/auth.rb"
database = Sequel.connect(ENV.fetch("DATABASE_URL"))

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  database: BetterAuth::Adapters::MSSQL.new(
    connection: database
  )
)
```

## Schema

The Ruby schema maps Better Auth logical fields to MSSQL-friendly physical
columns. String columns use `varchar(255)` where indexing is needed and
`varchar(8000)` for larger text values.

| Better Auth field | MSSQL column |
| --- | --- |
| `emailVerified` | `email_verified` |
| `userId` | `user_id` |
| `createdAt` | `created_at` |
| `expiresAt` | `expires_at` |

Plugin schemas are merged into the same SQL generation path.
