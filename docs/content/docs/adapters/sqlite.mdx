---
title: SQLite
description: Use Better Auth Ruby with SQLite.
---

Better Auth Ruby includes a direct SQLite adapter for local development,
single-node Rack apps, tests, and small deployments.

<Callout type="info">
SQLite support is implemented through the `sqlite3` gem. The adapter uses the
same Better Auth schema metadata as PostgreSQL and MySQL, with SQLite-friendly
column types and foreign keys.
</Callout>

## Install Dependencies

```ruby title="Gemfile"
gem "better_auth"
gem "sqlite3"
```

```bash title="Terminal"
bundle install
```

## Configure The Adapter

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL"),
  database: BetterAuth::Adapters::SQLite.new(
    path: "storage/better_auth.sqlite3"
  )
)
```

You may also pass an existing `SQLite3::Database` object when your application
owns connection lifecycle.

```ruby title="config/auth.rb"
database = SQLite3::Database.new("storage/better_auth.sqlite3")
database.results_as_hash = true

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  database: BetterAuth::Adapters::SQLite.new(
    connection: database
  )
)
```

## Schema

The adapter stores Better Auth logical fields as `snake_case` SQLite columns.
Foreign keys are enabled by the adapter with `PRAGMA foreign_keys = ON`.

| Better Auth field | SQLite column |
| --- | --- |
| `emailVerified` | `email_verified` |
| `userId` | `user_id` |
| `createdAt` | `created_at` |
| `expiresAt` | `expires_at` |

Plugin schemas are merged into the same SQL generation path.
