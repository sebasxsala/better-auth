---
title: MongoDB
description: Use Better Auth Ruby with MongoDB.
---

Better Auth Ruby includes a standalone MongoDB adapter for applications that
want document storage instead of SQL.

<Callout type="info">
MongoDB support maps Better Auth's logical `id` field to Mongo `_id` and stores
default collections with upstream-compatible singular names such as `user`,
`session`, `account`, and `verification`.
</Callout>

## Install Dependencies

```ruby title="Gemfile"
gem "better_auth"
gem "mongo"
```

```bash title="Terminal"
bundle install
```

## Configure The Adapter

```ruby title="config/auth.rb"
require "better_auth"
require "mongo"

client = Mongo::Client.new(
  ENV.fetch("MONGODB_URL", "mongodb://127.0.0.1:27017/better_auth")
)

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL"),
  database: BetterAuth::Adapters::MongoDB.new(
    database: client.database,
    client: client
  )
)
```

Transactions are enabled when a client is provided. If your MongoDB deployment
does not support transactions, disable them explicitly.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  database: BetterAuth::Adapters::MongoDB.new(
    database: client.database,
    client: client,
    transaction: false
  )
)
```

## Schema

MongoDB does not need SQL schema generation or migrations. The adapter writes
documents using Better Auth storage field names.

| Better Auth field | MongoDB field |
| --- | --- |
| `id` | `_id` |
| `emailVerified` | `email_verified` |
| `userId` | `user_id` |
| `createdAt` | `created_at` |
| `expiresAt` | `expires_at` |

The adapter supports the same core CRUD contract as the SQL adapters, including
where operators, sorting, pagination, counts, and core joins used by sessions
and accounts.
