---
title: CLI
description: Command-line tooling for Better Auth Ruby.
---

Better Auth Ruby does not use the upstream Node CLI. Ruby projects use Ruby and
framework-native commands.

## Rails Install Generator

Create the initializer and base migration:

```bash title="Terminal"
bin/rails generate better_auth:install
```

The same setup is available as a Rails task:

```bash title="Terminal"
bin/rails better_auth:init
```

## Rails Migration Generator

Generate only Better Auth migration files:

```bash title="Terminal"
bin/rails generate better_auth:migration
```

or:

```bash title="Terminal"
bin/rails better_auth:generate:migration
```

The generators skip an existing initializer or existing
`*_create_better_auth_tables.rb` migration instead of overwriting it.

## Core Gem Development Commands

Inside `packages/better_auth`:

```bash title="Terminal"
bundle install
bundle exec rake test
bundle exec standardrb
bundle exec rake ci
```

The package also exposes Make targets:

```bash title="Terminal"
make help
make test
make lint
make ci
```

## Rails Adapter Development Commands

Inside `packages/better_auth-rails`:

```bash title="Terminal"
bundle install
bundle exec rspec
bundle exec standardrb
```

## Not Supported

- `npx @better-auth/cli` commands.
- Upstream CLI commands for generating TypeScript config, client files, or Node
  migrations.
- A standalone Ruby CLI for non-Rails migrations. Rack apps should use their own
  migration tooling and the schema APIs from the core gem.
