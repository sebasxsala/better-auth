---
title: Cookies
description: Learn how cookies are used in Better Auth Ruby.
---

Better Auth Ruby uses signed cookies for session tokens, session cache data,
OAuth account cache data, and the `dont_remember` flag. All signing uses the
configured `secret`.

## Cookie Names

By default, cookies use the `better-auth` prefix:

- `better-auth.session_token`
- `better-auth.session_data`
- `better-auth.account_data`
- `better-auth.dont_remember`

When secure cookies are enabled, the runtime prefixes cookie names with
`__Secure-`.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  advanced: {
    cookie_prefix: "my-app"
  }
)
```

## Cookie Attributes

Cookies are `http_only`, `same_site: "lax"`, and scoped to `/` by default.
They are marked secure when `advanced.use_secure_cookies` is true, when
`base_url` starts with `https://`, or when the app runs in production.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  base_url: "https://auth.example.com",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  advanced: {
    use_secure_cookies: true,
    default_cookie_attributes: {
      same_site: "none"
    }
  }
)
```

Override one cookie through `advanced.cookies`.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  advanced: {
    cookies: {
      session_token: {
        name: "custom_session_token",
        attributes: {
          same_site: "lax",
          path: "/"
        }
      }
    }
  }
)
```

## Cross-Subdomain Cookies

Enable cross-subdomain cookies only when the auth server and application share a
trusted parent domain.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  base_url: "https://auth.example.com",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  advanced: {
    cross_subdomain_cookies: {
      enabled: true,
      domain: "example.com"
    }
  },
  trusted_origins: [
    "https://app.example.com"
  ]
)
```

<Callout type="warn">
Do not set a broad cookie domain unless every subdomain in that scope is trusted.
</Callout>

## Session Cookie Cache

Session cookie cache stores a short-lived copy of the parsed session response in
`session_data`.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  session: {
    cookie_cache: {
      enabled: true,
      max_age: 5 * 60,
      strategy: "jwe"
    }
  }
)
```

Supported strategies are `jwe`, `jwt`, and `compact`.

## Account Cookie Cache

OAuth account data can be cached in `account_data` when
`account.store_account_cookie` is enabled.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  account: {
    store_account_cookie: true
  }
)
```

## Safari, ITP, And Cross-Domain Setups

Safari can block cookies when the frontend and auth API are on unrelated
domains.

```txt
Frontend: https://app.domain-b.com
API:      https://auth.domain-a.com
```

Prefer proxying `/api/auth/*` through the frontend's site, or use a shared parent
domain such as `app.example.com` and `auth.example.com` with carefully scoped
cross-subdomain cookies.

## Not Supported

- Browser-side cookie helpers from upstream JavaScript packages.
- TypeScript deployment snippets. Use your Ruby framework or hosting provider's
  native reverse proxy configuration.
