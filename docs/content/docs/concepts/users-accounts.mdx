---
title: User & Accounts
description: User and account management.
---

Better Auth Ruby stores user profile data in the `user` table and authentication
provider data in the `account` table.

## Users

The user record includes `id`, `name`, `email`, `emailVerified`, `image`,
`createdAt`, and `updatedAt`.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  user: {
    additional_fields: {
      role: {
        type: "string",
        required: false,
        default_value: "user"
      }
    }
  }
)
```

```ruby title="app/services/update_user.rb"
auth.api.update_user(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    name: "Jane Doe",
    image: "https://example.com/avatar.png",
    role: "admin"
  }
)
```

Email cannot be changed with `update_user`; use `change_email`.

## Change Email

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  user: {
    change_email: {
      enabled: true,
      update_email_without_verification: false
    }
  },
  email_verification: {
    send_verification_email: ->(data, _request) {
      UserMailer.verify_email(data).deliver_later
    }
  }
)
```

```ruby title="app/services/change_email.rb"
auth.api.change_email(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    new_email: "new-email@example.com"
  }
)
```

## Passwords

```ruby title="app/services/passwords.rb"
auth.api.set_password(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    new_password: "new-password123"
  }
)

auth.api.verify_password(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    password: "password123"
  }
)

auth.api.change_password(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    current_password: "password123",
    new_password: "new-password123",
    revoke_other_sessions: true
  }
)
```

## Delete User

Deletion must be explicitly enabled.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  user: {
    delete_user: {
      enabled: true,
      before_delete: ->(user, _request) {
        AuditLog.create!(event: "user_delete_requested", user_id: user.fetch("id"))
      },
      after_delete: ->(user, _request) {
        AuditLog.create!(event: "user_deleted", user_id: user.fetch("id"))
      }
    }
  }
)
```

```ruby title="app/services/delete_user.rb"
auth.api.delete_user(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    password: "password123"
  }
)
```

## Accounts

An account stores provider-specific authentication data: `accountId`,
`providerId`, `userId`, OAuth tokens, expiration dates, and credential password
hashes.

```ruby title="app/services/accounts.rb"
accounts = auth.api.list_accounts(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  }
)
```

## OAuth Token Encryption

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  account: {
    encrypt_oauth_tokens: true
  }
)
```

You can also use database hooks if you need custom encryption.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  database_hooks: {
    account: {
      create: {
        before: lambda do |account, _context|
          encrypted = account.dup
          encrypted["accessToken"] = encrypt(encrypted["accessToken"]) if encrypted["accessToken"]
          encrypted["refreshToken"] = encrypt(encrypted["refreshToken"]) if encrypted["refreshToken"]

          {
            data: encrypted
          }
        end
      }
    }
  }
)
```

## Account Linking

Account linking is enabled by default when the provider confirms the email as
verified.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  account: {
    account_linking: {
      enabled: true,
      trusted_providers: ["google", "github"],
      allow_different_emails: false,
      update_user_info_on_link: true
    }
  }
)
```

```ruby title="app/services/link_account.rb"
result = auth.api.link_social(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    provider: "google",
    callback_url: "/settings/accounts"
  }
)
```

```ruby title="app/services/unlink_account.rb"
auth.api.unlink_account(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    provider_id: "google",
    account_id: "provider-account-id"
  }
)
```

By default, Better Auth prevents unlinking the user's last account.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  account: {
    account_linking: {
      allow_unlinking_all: true
    }
  }
)
```

## Not Supported

- Browser client account helpers in these examples. Use HTTP routes or
  `auth.api` from Ruby.
- TypeScript account and user inference examples.
