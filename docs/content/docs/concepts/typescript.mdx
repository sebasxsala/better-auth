---
title: Ruby Port Notes
description: Ruby-specific notes for Better Auth.
---

The Ruby port follows upstream Better Auth behavior where possible, but the
public API is Ruby-first.

## Configuration Names

Use `snake_case` option names in Ruby.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  base_url: "https://app.example.com/api/auth",
  base_path: "/api/auth",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  email_and_password: {
    enabled: true,
    min_password_length: 8,
    max_password_length: 128
  },
  social_providers: {
    github: BetterAuth::SocialProviders.github(
      client_id: ENV.fetch("GITHUB_CLIENT_ID"),
      client_secret: ENV.fetch("GITHUB_CLIENT_SECRET")
    )
  }
)
```

Request bodies may use either upstream-style camelCase keys or Ruby-style
snake_case keys. Internally, Better Auth maps them to upstream logical field
names.

```ruby title="app/services/auth.rb"
auth.api.sign_up_email(
  body: {
    email: "jane@example.com",
    password: "password123",
    name: "Jane",
    callback_url: "/dashboard"
  }
)
```

## Runtime Return Values

Endpoint responses are Ruby hashes and arrays.

```ruby title="app/services/session_lookup.rb"
session = auth.api.get_session(
  headers: {
    "cookie" => cookie_header
  }
)

user_id = session.fetch(:user).fetch("id")
```

Some nested values use upstream logical field names, such as `emailVerified`,
`userId`, and `providerId`, because they mirror the Better Auth schema.

## Errors

Endpoint failures raise `BetterAuth::APIError` in server-side Ruby calls.

```ruby title="app/services/sign_in.rb"
begin
  auth.api.sign_in_email(
    body: {
      email: "jane@example.com",
      password: "wrong-password"
    }
  )
rescue BetterAuth::APIError => error
  [error.status_code, error.code, error.message]
end
```

## Rails Integration

Rails apps should configure Better Auth in
`config/initializers/better_auth.rb`.

```ruby title="config/initializers/better_auth.rb"
BetterAuth::Rails.configure do |config|
  config.secret =
    Rails.application.credentials.dig(:better_auth, :secret) ||
    Rails.application.credentials.secret_key_base ||
    Rails.application.secret_key_base

  config.base_url = ENV["BETTER_AUTH_URL"]
  config.base_path = "/api/auth"
end
```

Mount routes with:

```ruby title="config/routes.rb"
Rails.application.routes.draw do
  better_auth
end
```

## Not Supported

- TypeScript type inference.
- `InferSession`, `InferUser`, and other upstream type helper examples.
- `tsconfig.json` guidance.
- Generated TypeScript clients from the Ruby docs.
