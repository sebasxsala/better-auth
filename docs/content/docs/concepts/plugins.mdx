---
title: Plugins
description: Learn how to use plugins with Better Auth Ruby.
---

Plugins extend Better Auth with endpoints, schema, hooks, middleware, rate
limits, and request/response handlers.

## Using A Plugin

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  plugins: [
    BetterAuth::Plugins.admin,
    BetterAuth::Plugins.api_key
  ]
)
```

Rails apps configure plugins in the initializer.

```ruby title="config/initializers/better_auth.rb"
BetterAuth::Rails.configure do |config|
  config.plugins = [
    BetterAuth::Plugins.admin,
    BetterAuth::Plugins.api_key
  ]
end
```

Plugin schemas are included when Rails migrations are generated.

```bash title="Terminal"
bin/rails generate better_auth:migration
```

## Custom Plugin

```ruby title="config/auth.rb"
hello_plugin = BetterAuth::Plugin.new(
  id: "hello",
  endpoints: {
    say_hello: BetterAuth::Endpoint.new(
      path: "/hello",
      method: "GET"
    ) do |ctx|
      ctx.json(
        message: "Hello from #{ctx.context.options.app_name}"
      )
    end
  }
)

auth = BetterAuth.auth(
  app_name: "Example App",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  plugins: [hello_plugin]
)
```

The endpoint is available over HTTP and through `auth.api`.

```ruby title="app/services/hello.rb"
auth.api.say_hello
```

## Plugin Schema

```ruby title="config/auth.rb"
invites_plugin = BetterAuth::Plugin.new(
  id: "invites",
  schema: {
    invite: {
      model_name: "invites",
      fields: {
        email: {
          type: "string",
          required: true
        },
        invitedBy: {
          type: "string",
          required: true,
          field_name: "invited_by"
        }
      }
    }
  }
)
```

Schema keys stay Better Auth-compatible. SQL adapters can map physical names
with `field_name`.

## Plugin Hooks

```ruby title="config/auth.rb"
audit_plugin = BetterAuth::Plugin.new(
  id: "audit",
  hooks: {
    after: [
      {
        matcher: ->(ctx) { ctx.path == "/sign-in/email" },
        handler: ->(ctx) {
          AuditLog.create!(event: "sign_in_email", payload: ctx.returned)
          nil
        }
      }
    ]
  }
)
```

## Plugin Middleware

Middleware can intercept matching Better Auth paths before the endpoint runs.

```ruby title="config/auth.rb"
internal_plugin = BetterAuth::Plugin.new(
  id: "internal-auth",
  middlewares: [
    {
      path: "/internal/**",
      middleware: lambda do |ctx|
        next if ctx.headers["x-internal-token"] == ENV["INTERNAL_AUTH_TOKEN"]

        [
          403,
          {"content-type" => "application/json"},
          [{error: "Forbidden"}.to_json]
        ]
      end
    }
  ]
)
```

## Request And Response Handlers

```ruby title="config/auth.rb"
headers_plugin = BetterAuth::Plugin.new(
  id: "headers",
  on_response: lambda do |response, _context|
    status, headers, body = response
    headers["x-powered-by"] = "better-auth-ruby"

    {
      response: [status, headers, body]
    }
  end
)
```

## Plugin Rate Limits

```ruby title="config/auth.rb"
limited_plugin = BetterAuth::Plugin.new(
  id: "limited",
  rate_limit: [
    {
      path_matcher: ->(path) { path == "/hello" },
      window: 60,
      max: 10
    }
  ]
)
```

## Plugin Defaults

Plugins can define `init` to merge default options into the auth configuration.

```ruby title="config/auth.rb"
defaults_plugin = BetterAuth::Plugin.new(
  id: "defaults",
  init: lambda do |_context|
    {
      options: {
        session: {
          cookie_cache: {
            enabled: true,
            max_age: 5 * 60
          }
        }
      }
    }
  end
)
```

## Not Supported

- Client plugins, client atoms, and framework hooks from upstream JavaScript
  packages.
- TypeScript plugin inference helpers.
