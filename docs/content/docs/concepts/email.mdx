---
title: Email
description: Learn how to use email with Better Auth Ruby.
---

Better Auth Ruby does not send email by itself. You provide Ruby callables for
verification, password reset, and account deletion emails.

## Email Verification

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  base_url: "https://app.example.com/api/auth",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  email_and_password: {
    enabled: true,
    require_email_verification: true
  },
  email_verification: {
    send_on_sign_up: true,
    send_on_sign_in: true,
    expires_in: 60 * 60,
    send_verification_email: lambda do |data, request|
      UserMailer.with(
        user: data.fetch(:user),
        url: data.fetch(:url),
        token: data.fetch(:token),
        ip: request&.ip
      ).verify_email.deliver_later
    end
  }
)
```

The verification payload includes `:user`, `:url`, and `:token`.

Send a verification email manually:

```ruby title="app/services/email_verification.rb"
auth.api.send_verification_email(
  body: {
    email: "jane@example.com",
    callback_url: "/dashboard"
  },
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  }
)
```

Verify an email token:

```ruby title="app/controllers/email_verifications_controller.rb"
auth.api.verify_email(
  query: {
    token: params.fetch(:token),
    callback_url: "/dashboard"
  },
  as_response: true
)
```

## Verification Hooks

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  email_verification: {
    send_verification_email: ->(data, _request) {
      UserMailer.verify_email(data).deliver_later
    },
    before_email_verification: ->(user, _request) {
      Rails.logger.info("Verifying #{user.fetch('email')}")
    },
    on_email_verification: ->(user, _request) {
      Analytics.track("email_verification_started", user_id: user.fetch("id"))
    },
    after_email_verification: ->(user, _request) {
      Analytics.track("email_verified", user_id: user.fetch("id"))
    },
    auto_sign_in_after_verification: true
  }
)
```

## Password Reset

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  base_url: "https://app.example.com/api/auth",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  email_and_password: {
    enabled: true,
    send_reset_password: lambda do |data, request|
      UserMailer.with(
        user: data.fetch(:user),
        url: data.fetch(:url),
        token: data.fetch(:token),
        ip: request&.ip
      ).reset_password.deliver_later
    end,
    reset_password_token_expires_in: 60 * 60,
    revoke_sessions_on_password_reset: true,
    on_password_reset: ->(data, _request) {
      SecurityLog.password_reset(data.fetch(:user).fetch("id"))
    }
  }
)
```

```ruby title="app/services/password_reset.rb"
auth.api.request_password_reset(
  body: {
    email: "jane@example.com",
    redirect_to: "/reset-password"
  }
)

auth.api.reset_password(
  body: {
    token: params.fetch(:token),
    new_password: "new-password123"
  }
)
```

## Change Email

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  user: {
    change_email: {
      enabled: true,
      update_email_without_verification: false
    }
  },
  email_verification: {
    send_verification_email: ->(data, _request) {
      UserMailer.verify_email(data).deliver_later
    }
  }
)
```

```ruby title="app/services/change_email.rb"
auth.api.change_email(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    new_email: "new-email@example.com"
  }
)
```

## Delete Account Email

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  user: {
    delete_user: {
      enabled: true,
      delete_token_expires_in: 60 * 60,
      send_delete_account_verification: ->(data, _request) {
        UserMailer.delete_account(data).deliver_later
      }
    }
  }
)
```

## Not Supported

- Built-in email delivery adapters. Use Action Mailer, Mail, or your delivery
  layer.
- Upstream browser client helpers in this page. Call HTTP endpoints or
  server-side `auth.api` methods from Ruby.
