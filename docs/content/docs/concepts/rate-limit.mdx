---
title: Rate Limit
description: How to limit the number of requests a user can make to the server in a given time period.
---

Better Auth Ruby includes a Rack-level rate limiter. It runs for
client-initiated HTTP requests that pass through the Better Auth router.

<Callout type="warning">
Server-side requests made through `auth.api` are not affected by rate limiting.
Rate limits apply to HTTP requests handled by the Rack router. If a public
controller, job, or RPC method calls `auth.api` using user input, rate limit that
public entry point separately.
</Callout>

## Defaults

Rate limiting is enabled automatically in production environments and disabled
by default in development and test.

The Ruby defaults are:

- Window: `10` seconds
- Max requests: `100`
- Storage: memory, unless `secondary_storage` is configured

Sensitive core paths get a stricter `3` requests per `10` seconds:

- `/sign-in/*`
- `/sign-up/*`
- `/change-password`
- `/change-email`

## Configure Rate Limit

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  rate_limit: {
    enabled: true,
    window: 60,
    max: 100
  }
)
```

Rails apps use the same options in the initializer.

```ruby title="config/initializers/better_auth.rb"
BetterAuth::Rails.configure do |config|
  config.rate_limit = {
    enabled: true,
    window: 60,
    max: 100
  }
end
```

## IP Address

Rate limiting uses the request IP address. Configure trusted proxy headers with
`advanced.ip_address.ip_address_headers`.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  advanced: {
    ip_address: {
      ip_address_headers: ["cf-connecting-ip", "x-forwarded-for"],
      disable_ip_tracking: false
    }
  },
  rate_limit: {
    enabled: true,
    window: 60,
    max: 100
  }
)
```

## Custom Rules

Use `custom_rules` to override limits for specific paths.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  rate_limit: {
    enabled: true,
    custom_rules: {
      "/sign-in/email" => {
        window: 10,
        max: 3
      },
      "/two-factor/*" => lambda do |_request, current_rule|
        {
          window: current_rule[:window],
          max: 3
        }
      end
    }
  }
)
```

Set a rule to `false`, or return `false` from a rule callable, to disable rate
limiting for that path.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  rate_limit: {
    enabled: true,
    custom_rules: {
      "/get-session" => false
    }
  }
)
```

Plugins can provide rate limit rules with a `path_matcher`.

```ruby title="config/auth.rb"
plugin = BetterAuth::Plugin.new(
  id: "strict-password-reset",
  rate_limit: [
    {
      path_matcher: ->(path) { path == "/request-password-reset" },
      window: 60,
      max: 2
    }
  ]
)
```

## Storage

Memory storage is the default. If `secondary_storage` is configured, counters can
be stored there.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  secondary_storage: redis_storage,
  rate_limit: {
    enabled: true,
    storage: "secondary-storage"
  }
)
```

Custom storage is also supported.

```ruby title="config/auth.rb"
class RateLimitStore
  def get(key)
    Redis.current.get(key)
  end

  def set(key, value, ttl:, update: false)
    Redis.current.set(key, value.to_json, ex: ttl)
  end
end

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  rate_limit: {
    enabled: true,
    custom_storage: RateLimitStore.new
  }
)
```

## Handling Errors

When the limit is exceeded, Better Auth returns HTTP `429`, the
`x-retry-after` header, and a JSON body with `Too many requests. Please try
again later.`

## Not Supported

- Database-backed runtime rate limit storage. Use memory, `secondary_storage`,
  or `custom_storage`.
- Upstream JavaScript client error handling examples.
