---
title: Session Management
description: Better Auth Ruby session management.
---

Better Auth Ruby uses cookie-based sessions. The session token is stored in a
signed cookie and the server resolves the token on authenticated requests.

## Session Table

The session record stores `id`, `token`, `userId`, `expiresAt`, `ipAddress`,
`userAgent`, `createdAt`, and `updatedAt`.

## Session Expiration

Sessions expire after seven days by default.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  session: {
    expires_in: 60 * 60 * 24 * 7,
    update_age: 60 * 60 * 24,
    fresh_age: 60 * 60 * 24
  }
)
```

`fresh_age` protects sensitive endpoints such as password changes and account
deletion. Set it to `0` to disable freshness checks.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  session: {
    fresh_age: 0
  }
)
```

## Get Session

```ruby title="app/controllers/application_controller.rb"
session = auth.api.get_session(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  }
)

current_user = session&.fetch(:user)
```

Disable cookie cache or refresh for one request:

```ruby title="app/services/session_lookup.rb"
auth.api.get_session(
  headers: {
    "cookie" => cookie_header
  },
  query: {
    disable_cookie_cache: true,
    disable_refresh: true
  }
)
```

## List And Revoke Sessions

```ruby title="app/services/session_management.rb"
headers = {
  "cookie" => request.get_header("HTTP_COOKIE").to_s
}

sessions = auth.api.list_sessions(headers: headers)

auth.api.revoke_session(
  headers: headers,
  body: {
    token: sessions.first.fetch("token")
  }
)

auth.api.revoke_other_sessions(headers: headers)
auth.api.revoke_sessions(headers: headers)
```

## Sign Out

```ruby title="app/controllers/sessions_controller.rb"
status, headers, body = auth.api.sign_out(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  as_response: true
)
```

## Change Password

```ruby title="app/services/change_password.rb"
auth.api.change_password(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    current_password: "old-password123",
    new_password: "new-password123",
    revoke_other_sessions: true
  }
)
```

## Cookie Cache

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  session: {
    cookie_cache: {
      enabled: true,
      max_age: 5 * 60,
      strategy: "jwe",
      refresh_cache: false
    }
  }
)
```

Supported strategies are `jwe`, `jwt`, and `compact`.

## Stateless Sessions

When no database is configured, Better Auth Ruby enables stateless defaults that
use encrypted cookie cache data.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  database: nil,
  session: {
    cookie_cache: {
      enabled: true,
      strategy: "jwe"
    }
  },
  account: {
    store_account_cookie: true
  }
)
```

## Secondary Storage

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  secondary_storage: redis_storage,
  session: {
    store_session_in_database: false
  }
)
```

## Session Fields

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  session: {
    additional_fields: {
      active_organization_id: {
        type: "string",
        required: false,
        field_name: "active_organization_id"
      }
    }
  }
)
```

```ruby title="app/services/session_update.rb"
auth.api.update_session(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    active_organization_id: "org_123"
  }
)
```

## Not Supported

- Browser session hooks such as framework-specific `useSession` helpers.
- TypeScript custom session inference examples. Use Ruby hashes returned from
  `auth.api.get_session`.
