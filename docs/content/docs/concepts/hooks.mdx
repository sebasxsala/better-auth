---
title: Hooks
description: Better Auth Ruby hooks let you customize Better Auth behavior.
---

Better Auth Ruby supports API hooks around endpoint execution and database hooks
around adapter writes.

## API Hooks

API hooks run for both Rack requests and direct `auth.api` calls.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  hooks: {
    before: lambda do |ctx|
      next unless ctx.path == "/sign-up/email"

      ctx.body["source"] = "web"

      {
        context: {
          body: ctx.body
        }
      }
    end
  }
)
```

Return a response hash to short-circuit the endpoint.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  hooks: {
    before: lambda do |ctx|
      next unless ctx.path == "/sign-in/email"
      next unless maintenance_mode?

      {
        error: "Sign in is temporarily unavailable"
      }
    end
  }
)
```

Use an `after` hook to replace or decorate the response.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  hooks: {
    after: lambda do |ctx|
      next unless ctx.path == "/get-session"
      next unless ctx.returned

      ctx.returned.merge(
        server_time: Time.now.utc.iso8601
      )
    end
  }
)
```

## Endpoint Context

Hook callbacks receive `path`, `method`, `query`, `body`, `params`, `headers`,
`request`, `context`, and `returned`. Hooks can set response headers and cookies
through context helpers.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  hooks: {
    after: lambda do |ctx|
      ctx.set_header("x-auth-path", ctx.path)
      nil
    end
  }
)
```

## Database Hooks

Database hooks run around adapter `create`, `update`, and `delete` operations.
They can transform write data or cancel deletes.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  database_hooks: {
    account: {
      create: {
        before: lambda do |account, _context|
          data = account.dup
          data["accessToken"] = encrypt(data["accessToken"]) if data["accessToken"]
          data["refreshToken"] = encrypt(data["refreshToken"]) if data["refreshToken"]

          {
            data: data
          }
        end
      }
    }
  }
)
```

`before` hooks can return `{ data: updated_data }`, `false` to cancel a delete,
or `nil` to continue unchanged.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  database_hooks: {
    user: {
      create: {
        after: ->(user, _context) {
          Analytics.track("user_created", user_id: user.fetch("id"))
        }
      }
    }
  }
)
```

## Plugin Hooks

```ruby title="config/auth.rb"
plugin = BetterAuth::Plugin.new(
  id: "audit",
  hooks: {
    after: [
      {
        matcher: ->(ctx) { ctx.path == "/sign-in/email" },
        handler: ->(ctx) {
          AuditLog.create!(
            event: "sign_in_email",
            response: ctx.returned
          )
          nil
        }
      }
    ]
  }
)

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  plugins: [plugin]
)
```

## Not Supported

- TypeScript hook builders or inference helpers.
- Browser/client hooks from upstream packages.
