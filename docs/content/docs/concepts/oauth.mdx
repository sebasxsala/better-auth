---
title: OAuth
description: How Better Auth Ruby handles OAuth.
---

Better Auth Ruby includes OAuth 2.0 and OpenID Connect flows for supported
social providers. Configure providers under `social_providers`.

## Configure Social Providers

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  base_url: "https://app.example.com/api/auth",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  social_providers: {
    google: BetterAuth::SocialProviders.google(
      client_id: ENV.fetch("GOOGLE_CLIENT_ID"),
      client_secret: ENV.fetch("GOOGLE_CLIENT_SECRET")
    ),
    github: BetterAuth::SocialProviders.github(
      client_id: ENV.fetch("GITHUB_CLIENT_ID"),
      client_secret: ENV.fetch("GITHUB_CLIENT_SECRET")
    )
  }
)
```

The OAuth callback path is derived from `base_url`:

```txt
https://app.example.com/api/auth/callback/google
```

## Sign In

Start a redirect-based sign-in.

```ruby title="app/controllers/oauth_controller.rb"
result = auth.api.sign_in_social(
  body: {
    provider: "google",
    callback_url: "/dashboard"
  }
)

redirect_to result.fetch(:url), allow_other_host: true
```

To get the URL without an automatic redirect flag:

```ruby title="app/services/oauth.rb"
result = auth.api.sign_in_social(
  body: {
    provider: "google",
    callback_url: "/dashboard",
    disable_redirect: true
  }
)

authorization_url = result.fetch(:url)
```

## ID Token Sign In

Providers that implement ID token verification can sign in without a redirect.

```ruby title="app/services/native_google_sign_in.rb"
result = auth.api.sign_in_social(
  body: {
    provider: "google",
    id_token: {
      token: "id-token-from-provider",
      nonce: "nonce-used-for-token",
      access_token: "access-token",
      refresh_token: "refresh-token"
    }
  }
)
```

## Link Social Accounts

```ruby title="app/controllers/account_links_controller.rb"
result = auth.api.link_social(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    provider: "google",
    callback_url: "/settings/accounts",
    scopes: ["https://www.googleapis.com/auth/drive.readonly"]
  }
)

redirect_to result.fetch(:url), allow_other_host: true
```

Linking with an ID token is also supported when the provider supports ID token
verification.

```ruby title="app/services/link_native_account.rb"
auth.api.link_social(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    provider: "google",
    id_token: {
      token: "id-token-from-provider",
      access_token: "access-token"
    }
  }
)
```

## Access Tokens

Get an access token for a linked account. If the stored token is expired and the
provider can refresh it, Better Auth refreshes and stores the new token.

```ruby title="app/services/google_drive_token.rb"
token = auth.api.get_access_token(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  body: {
    provider_id: "google"
  }
)

access_token = token.fetch(:accessToken)
```

Server-side jobs can pass `user_id` when no session cookie is available.

```ruby title="app/jobs/sync_google_drive_job.rb"
token = auth.api.get_access_token(
  body: {
    user_id: user.id,
    provider_id: "google"
  }
)
```

Refresh a token explicitly:

```ruby title="app/services/oauth_refresh.rb"
auth.api.refresh_token(
  body: {
    user_id: user.id,
    provider_id: "google"
  }
)
```

## Provider Account Info

```ruby title="app/services/oauth_account_info.rb"
info = auth.api.account_info(
  headers: {
    "cookie" => request.get_header("HTTP_COOKIE").to_s
  },
  query: {
    account_id: params.fetch(:account_id)
  }
)
```

## Account Linking Rules

Account linking is enabled by default when the provider confirms the email as
verified.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  account: {
    account_linking: {
      enabled: true,
      trusted_providers: ["google", "github"],
      allow_different_emails: false,
      update_user_info_on_link: true
    }
  }
)
```

## Token Encryption

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  account: {
    encrypt_oauth_tokens: true
  }
)
```

## Not Supported

- JavaScript client OAuth helpers in these concept examples. Use HTTP routes
  from the frontend or call `auth.api` from Ruby.
- OAuth provider behavior that depends on a provider method not implemented by
  the configured Ruby provider, such as ID token verification for providers that
  do not expose `verify_id_token`.
