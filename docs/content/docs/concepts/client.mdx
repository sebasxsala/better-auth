---
title: Client
description: Better Auth Ruby client integration.
---

Better Auth Ruby exposes the Better Auth HTTP route surface from a Rack app.
Frontend and non-Ruby clients should call those HTTP routes.

There is no Ruby browser client package. Server-side Ruby code should use
`auth.api`; browser code should call the mounted auth routes.

## Mount The Auth Server

Rack:

```ruby title="config.ru"
require "better_auth"

auth = BetterAuth.auth(
  base_url: "http://localhost:9292/api/auth",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  email_and_password: {
    enabled: true
  }
)

map "/api/auth" do
  run auth
end
```

Rails:

```ruby title="config/routes.rb"
Rails.application.routes.draw do
  better_auth
end
```

## Sign Up

```bash title="Terminal"
curl -i http://localhost:3000/api/auth/sign-up/email \
  -H "content-type: application/json" \
  -d '{"email":"jane@example.com","password":"password123","name":"Jane"}'
```

The response includes a session cookie when auto sign-in is enabled.

## Sign In

```bash title="Terminal"
curl -i http://localhost:3000/api/auth/sign-in/email \
  -H "content-type: application/json" \
  -d '{"email":"jane@example.com","password":"password123"}'
```

Store the returned `set-cookie` value and send it on authenticated requests.

## Get Session

```bash title="Terminal"
curl http://localhost:3000/api/auth/get-session \
  -H "Cookie: better-auth.session_token=..."
```

## Sign Out

```bash title="Terminal"
curl -i http://localhost:3000/api/auth/sign-out \
  -X POST \
  -H "Cookie: better-auth.session_token=..."
```

## Server-Side Ruby Client

Use `auth.api` when Ruby code is calling Better Auth internally.

```ruby title="app/services/auth_client.rb"
result = auth.api.sign_in_email(
  return_headers: true,
  body: {
    email: "jane@example.com",
    password: "password123"
  },
  headers: {
    "user-agent" => "Ruby service"
  }
)

cookie = result[:headers]["set-cookie"]
```

## Rails Controller Helpers

Rails apps can use controller helpers instead of manually calling
`get_session`.

```ruby title="app/controllers/application_controller.rb"
class ApplicationController < ActionController::Base
  include BetterAuth::Rails::ControllerHelpers
end
```

```ruby title="app/controllers/posts_controller.rb"
class PostsController < ApplicationController
  before_action :require_authentication

  def index
    @user = current_user
  end
end
```

Available helpers:

- `current_session`
- `current_user`
- `authenticated?`
- `require_authentication`

## Cross-Origin Requests

If your browser client runs on another origin, configure `trusted_origins`.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  base_url: "https://auth.example.com/api/auth",
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  trusted_origins: [
    "https://app.example.com"
  ]
)
```

For cookie-based browser sessions, make sure requests include credentials and
that your framework or proxy allows credentials for the trusted origin.

## Not Supported

- A Ruby equivalent of the upstream browser client package.
- Framework-specific client hooks such as React, Vue, Svelte, or Solid helpers.
- TypeScript plugin client inference.
