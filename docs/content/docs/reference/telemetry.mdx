---
title: Telemetry
description: Telemetry behavior in Better Auth Ruby.
---

Better Auth Ruby does not collect or send telemetry.

The upstream JavaScript project has optional anonymous telemetry for Node-based runtime and CLI usage. That telemetry system has not been ported to Ruby, and the Ruby gems do not include a telemetry client, background sender, or `BETTER_AUTH_TELEMETRY` runtime integration.

## What Is Sent

Nothing is sent by Better Auth Ruby itself.

The Ruby port may still run inside applications that use their own analytics, logging, error reporting, APM, or hosting telemetry. Those systems are outside Better Auth Ruby and should be configured in your application or hosting provider.

## Environment Variables

These upstream JavaScript environment variables are not used by Better Auth Ruby:

- `BETTER_AUTH_TELEMETRY`
- `BETTER_AUTH_TELEMETRY_DEBUG`

Setting them has no effect in the Ruby gem.

## Logging Instead

Use `logger` when you want Better Auth Ruby warnings or errors routed into your application logger.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  logger: ->(level, message) {
    Rails.logger.public_send(level, message)
  }
)
```

The logger receives local runtime messages only. It does not send data to Better Auth maintainers.
